Chiến lược static website với GithubPages

Admin | 22-09-2017

Để tăng thêm sự hứng thú, thì tôi xin cung cấp thông tin là webiste này cũng được xây dựng trên chiến lược static web với GitHubPages.

Lợi thế của chiến lược này:

  • - Load cực nhanh và không thể bị hack (vì nó là web tĩnh, cache được hỗ trợ tối đa -> SEO cực tốt)
  • - Hoàn toàn free, không tốn phí duy trì. Nếu không dùng tên miền của github thì mới tốn thêm chi phí tên miền.
  • - Rất đơn giản để triển khai ở mức basic

Nhược điểm:

  • - Không thể áp dụng cho hệ thống phức tạp, cần tới ngôn ngữ back-end.
  • - Số lượng file html lớn, phải xây dựng bằng tay hoặc dùng Jekyll hoặc viết tool hỗ trợ generate ra file html.

Những việc cần chuẩn bị:

  • - HTML website, có 1 trang index.html, kiểu giống các HTML template
  • - Github account

Bước 1: Tạo Github repo

Repo được tạo với tên theo định dạng: username.github.io với username là username dùng để đăng nhập Github

Bước 2: Push source HTML website đã chuẩn bị lên repo

Command line để push source

git init
git add .
git commit -m "messages"
git remote link-to-repo
git push -u origin master

Bước 3: Check thành quả nào!

Truy cập vào url dạng username.github.io với username là username để đăng nhập Github

Việc hiển thị có thể mất khoảng 10 phút, vì đợi Github tiến hành build.

Gợi ý về việc triển khai với hệ thống phức tạp.

Chẳng hạn như blog hay danh mục sản phẩm, thì với chiến lược này, mỗi post cần phải có 1 trang html Vậy nên nếu không muốn làm bằng tay, chúng ta có thể dùng Jekyll (dùng generate ra html từ markdown). Tuy nhiên Jekyll chỉ có 1 số theme nhất định. Vì vậy ta có thể viết 1 tool để generate ra html, đọc file layout, replace nội dung như title, nội dung, ảnh ...

Các file html phải nằm cùng cấp trong file index.html, và đường link truy cập có dạng là: .../ten-file.html
